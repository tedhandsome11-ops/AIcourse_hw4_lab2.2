The Alchemist's Trial

1.Game Description
**「煉金術大師試煉」**是一款結合了角色扮演與知識互動的文字冒險遊戲。玩家扮演一名渴望獲得公會認可的煉金術學徒，離開安全的實驗室，深入遺跡與山林尋找古老的傳說配方。

遊戲的核心機制圍繞著煉金術的兩大支柱：

效能 (Potency)：代表對配方理論的精確掌握。玩家需透過「研讀理論」與導師進行問答互動來提升此數值。

調和度 (Harmony)：代表對煉金過程的藝術性與創造力。玩家需透過「進行實驗」輸入自定義的調配步驟，由系統生成實驗結果來提升此數值。

玩家的最終目標是在有限的行動次數內，將一種核心藥劑的「效能」修練至極致（解鎖精煉術），並將另一種藥劑的「調和度」提升至完美（創造簽名藥劑），最終獲得煉金術公會的認可。

2.Details on the LLM tasks applied
本專案運用了大型語言模型（GPT-3.5）來擔任遊戲管理員（GM）與導師的角色，並設計了多層次的生成任務，特別強調了「LLM 消化自身輸出」的機制：

  A. 知識合成與驗證 (Knowledge Synthesis & Verification)
  應用場景：玩家選擇「研讀配方理論」時。

  任務細節：

  LLM 讀取靜態的 JSON 配方資料（包含描述、副作用、傳說）。

  LLM 根據這些資料生成一個全新的理論問題。

  玩家回答後，LLM 再次根據原始資料評估玩家答案的正確性，並給予反饋。

  技術特點：模型必須理解結構化數據，並將其轉化為自然的對話問題，同時具備邏輯判斷能力。

  B. 創意敘事與自我精煉 (Creative Storytelling & Self-Correction)
  應用場景：玩家選擇「進行實驗」並輸入自定義動作時。

  任務細節：

  初步生成：LLM 根據玩家輸入的動作（如「加入龍鱗粉並快速攪拌」）生成一段短篇實驗結果故事。

  自我消化 (Self-Digestion)：系統將第一步生成的故事作為輸入，要求 LLM 「進行改寫與精煉」。指令包含特定約束（如：修正語氣、更合理的因果關係、禁止使用特定詞彙）。

  評分：最後由 LLM 判斷改寫後的故事是否符合「精妙的調和」標準，給予調和度分數。

  C. 摘要與事實查核 (Summarization & Fact-Checking)
  應用場景：遊戲結束生成「公會總評」時。

  任務細節：

  長文本摘要：LLM 讀取整場遊戲的完整 Log，將其壓縮為一份摘要報告。

  事實修正 (Truthfulness Refinement)：系統將「原始 Log（事實）」與「摘要草稿」同時提供給 LLM，要求其根據事實修正摘要，確保最終報告沒有幻覺（Hallucination）且語氣專業。這是最高層次的自我消化應用。

3. Discovery
在開發此專案的過程中，我有以下幾點發現：

透過要求 LLM 消化自己的輸出（例如在實驗描述中先生成初稿，再要求其改寫），最終產出的文本品質顯著提高。

我曾試過不提供 LLM 資料庫，純粹依賴 LLM 自由發揮，但容易導致內容發散或前後不一，且 LLM 引導玩家的敘述很容易模糊不清。當引入 JSON 格式的「配方資料庫」，讓 LLM 能有參考依據，生成的對話既有變化性又不失準確度。

角色扮演的沉浸感： LLM 非常擅長扮演特定的角色（如嚴格的導師或神祕的煉金筆記）。可以透過 Prompt 設定適當的語氣，能極大提升玩家的代入感，使文字遊戲不再只是枯燥的指令輸入。
